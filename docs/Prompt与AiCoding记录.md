# Prompt 与 Ai Coding 迭代记录

## 1. 记录说明

本文件记录项目中实际使用的关键输入、目标、生成结果、问题与调整，并对涉及身份和仓库账号的信息进行匿名化处理。

## P01：交付范围与硬性约束

### 原始输入摘要

> 选题为番茄钟；所有说明性文字使用 Markdown；项目说明使用 LaTeX 生成 PDF；PPT 不需要制作；界面采用包豪斯风格；代码使用远程仓库管理；2.2 的 Prompt、代码、调试和运行证据缺一不可；扩展任务选择同类产品三维横向对比。

### 目标

把分散要求转换为可执行清单，避免只完成代码而漏掉文档、截图、创新、扩展任务和版本管理。

### 结果与影响

生成任务总览，明确排除 PPT，建立源码、Markdown、LaTeX、证据和最终交付五条工作线。

## P02：技术创新收敛

### 输入

```text
为包豪斯风格浏览器番茄钟设计一个强但可在本周完成的创新点。
要求：可演示、可量化、可写入 W2 报告；必须有专属截图和自动测试；
优先考虑个性化体验与性能优化，避免把普通主题或音效包装成创新。
```

### 初始结果

提出“自适应专注节奏”和“抗后台节流精准计时”两个方向。

### 调整思路

没有把它们当成两个零散卖点，而是合并为“自适应专注节奏与精准计时引擎”：前者解释用户该用多久，后者保证这段时间可靠执行。

### 生成效果分析

加入可测条件后，输出不再停留在“智能推荐”口号，而是形成具体规则、上下界、冷却机制、误差指标和测试用例。结构化约束显著降低了后续返工。

## P03：界面设计方向

### 原始补充

用户提供两张包豪斯风格课程展示系统截图，并说明如果 Figma 无法读取，就直接采用这套风格。

### 视觉拆解 Prompt

```text
从参考截图中提取可复用的界面参数，不复制业务内容：
1. 背景、边框、阴影、主色；
2. 标题与小标签层级；
3. 网格与不对称布局；
4. 按钮和状态区的几何规则；
5. 移动端必须保留的功能。
将结果实现为集中 CSS 令牌和响应式组件。
```

### 结果

得到米白网格、四像素黑色硬边框、右下错位阴影、红黄蓝主色、超粗中文标题、英文全大写小标签和方形按钮，并写入 `styles.css` 与 `界面与交互设计.md`。

## P04：功能实现 Prompt

### 输入

```text
使用 React + TypeScript 实现番茄钟：
- 专注/休息、开始、暂停、继续、重置；
- 绝对结束时间作为计时事实来源；
- visibilitychange、focus 和刷新后重新校准；
- 当前任务、今日统计、最近记录、本地持久化；
- 三条记录后生成可解释建议；
- 建议限制 15–50 分钟并提供接受/拒绝；
- 逻辑拆为 Hook、纯函数与存储模块；
- 不引入组件库和状态管理库。
```

### 结果

生成 `usePrecisionTimer`、`recommendation.ts`、`storage.ts` 和四个展示组件。推荐算法与时间计算被拆成纯函数，可独立测试。

### 调整

初版只有完成记录，完成率会长期为 100%。在审查数据闭环后，增加“运行或暂停状态下重置即保存未完成样本”，使推荐算法能够真实观察中断行为。

## P05：测试与调试 Prompt

### 输入

```text
为推荐算法和绝对时间工具设计边界测试：
- 少于三条不推荐；
- 高完成率增加、低完成率或高暂停减少；
- 15–50 分钟边界；
- 两轮冷却；
- 剩余时间不为负；
- 毫秒到 MM:SS 的临界值。
然后运行类型检查、单元测试、生产构建和真实浏览器交互。
```

### 结果

完成 10 项单元测试；TypeScript 类型检查与 Vite 生产构建通过。真实浏览器完成任务输入、开始、暂停、重置、自适应建议触发和 390px 手机无横向滚动验证。

## 2. Prompt 方法总结

1. **先写边界再写功能**：明确“不做账号、云同步、PPT”，可以显著控制范围。
2. **把抽象词变成阈值**：“智能建议”改写为三条样本、完成率、暂停次数、上下界和冷却期。
3. **要求证据路径**：每个功能同时指定代码、测试和截图，减少“做了但无法证明”。
4. **一次只解决一个层级**：先架构与逻辑，再视觉，再浏览器验收，问题更容易定位。
5. **用失败结果驱动下一次 Prompt**：构建配置错误、沙箱限制和移动截图异常都记录真实现象后再调整，不重复无效指令。

## 3. AI 辅助编程的优势与局限

优势是能够快速把长要求拆成任务、并行完成竞品调研与审计、生成测试边界并保持文档与代码一致。局限是工具权限与环境差异会让“理论正确”的命令在本机失败；生成代码仍需真实类型检查、测试和浏览器观察，不能把模型回答当作完成证据。
