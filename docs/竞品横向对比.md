# 扩展任务：同类番茄钟产品横向对比

> 调研日期：2026 年 7 月 2 日  
> 调研原则：功能事实只引用产品官方页面；技术栈未公开时只做有限推断并明确标注。

## 1. 对比对象

- Forest：以树木生长和应用阻断形成游戏化专注体验。
- Focus To-Do（专注清单）：将番茄钟与任务、提醒和统计结合。
- Pomofocus：面向桌面和移动浏览器的轻量 Web 番茄钟。
- TickTick：将番茄钟置于待办、日历、习惯和统计组成的综合生产力系统中。

## 2. 三维对比

| 产品 | 功能完整性 | 用户体验分析 | 技术实现与边界 |
|---|---|---|---|
| Forest | 计时、应用阻断、可视化森林、专注统计、共同种树、环境声音和正念休息较完整。 | 把抽象时间转化为树木，反馈和约束力强；但社交、装饰与订阅层增加了纯计时用户的操作负担。 | 官方确认覆盖 iOS、Android、Apple Watch 与浏览器，并支持跨平台同步和核心离线使用。由此可推断其原生权限、账户和同步实现复杂度较高，但具体框架未公开。 |
| Focus To-Do | 番茄钟、任务、截止日期、提醒、子任务、重复、备注、报表和多设备同步形成完整闭环。 | “创建任务—专注—统计”路径清晰，适合学习计划和项目管理；功能入口比纯计时器多，首次使用成本更高。 | 官方确认支持 Android、iOS/iPad、Apple Watch、Mac、Windows 和 Chrome 扩展，并同步数据；官网未公开前后端技术栈。 |
| Pomofocus | 任务、预计番茄数、预计完成时间、模板、日周月报告、自定义时长和声音；高级版提供项目、年度报告、CSV、Todoist 与 Webhook。 | 网页打开即可使用，启动成本最低；高级项目管理与自动化能力需要付费。 | 官方确认在桌面与移动浏览器运行，并提供 Windows、macOS、Linux 下载。可以判断其以 Web 体验为核心，但官网未公开具体框架。 |
| TickTick | 番茄钟与待办、日历、习惯、看板、时间线、四象限、统计和跨端同步整合，功能范围最广。 | 计划和专注衔接紧密，但番茄钟只是大系统中的一个模块，信息密度与学习成本最高。 | 官方确认手机、电脑、平板和手表实时同步，并提供日历和第三方集成；具体实现技术未公开。 |
| 本项目 | 专注/休息、任务、历史、今日统计、精准计时、刷新恢复、自适应建议、响应式界面和几何成长反馈。 | 首屏聚焦当前任务和计时，最多两次操作即可开始；风格识别强，建议透明且可拒绝。当前版本没有复杂任务项目和跨设备同步。 | React + TypeScript + Vite；浏览器本地存储；绝对结束时间校准；纯函数推荐算法；无需账号和服务端。 |

## 3. 功能完整性结论

综合产品范围大致为 TickTick > Forest ≈ Focus To-Do > Pomofocus > 本项目。这个排序反映“覆盖功能数量”，不等于特定用户的体验优劣。本项目主动限制范围，不复制综合待办系统，而是把开发资源集中在计时准确性、低操作层级和个性化建议。

## 4. 用户体验结论

- Pomofocus 的 Web 启动成本最低。
- Forest 的成长反馈和情绪价值最强。
- Focus To-Do 的任务到复盘闭环最连贯。
- TickTick 的功能密度最高，适合已经使用其任务体系的用户。
- 本项目用强视觉识别、无账号和可解释建议形成差异，适合希望快速开始且关注隐私的用户。

## 5. 技术实现结论

Forest 需要原生应用阻断权限和跨平台账户同步，技术边界最宽；TickTick 与 Focus To-Do 体现成熟的多设备数据同步；Pomofocus 的 Web 优先路线更轻。本项目选择本地优先单页应用，部署与隐私边界清晰，但不提供跨设备同步。任何未由官方公开的 React、Vue、后端语言等信息均不写成事实。

## 6. 差异化策略

### 6.1 包豪斯几何成长反馈

每条完成记录参与生成当日几何海报，将圆、方、线组合成可视化积累。它吸收 Forest“让时间可见”的优点，但不复制种树题材。

### 6.2 可解释的自适应节奏

根据最近记录的完成率与暂停次数给出下一轮时长建议，并展示数据依据。用户可以接受或拒绝，避免黑箱式自动调整。

### 6.3 本地优先与隐私透明

项目无需登录，设置、计时状态和历史只保存在浏览器本机。界面直接说明“数据仅存本机”，把隐私边界做成可见的产品特征。

### 6.4 精准计时与刷新恢复

使用绝对结束时间，而不是依赖每秒回调累计。浏览器后台降频、切回页面或刷新后均重新计算剩余时间，提升纯 Web 番茄钟的可靠性。

### 6.5 克制的任务闭环

只保留当前任务、完成记录、今日统计和节奏建议，不复制 TickTick 式全能任务管理，减少上手成本。

## 7. 局限与后续方向

- 当前仅支持单设备本地数据，不适合需要跨端同步的用户。
- 未实现系统级应用阻断，约束力弱于 Forest。
- 自适应规则基于少量行为指标，不是机器学习模型；优点是可解释，缺点是个性化维度有限。
- 后续可在不破坏轻量定位的前提下增加数据导出、声音提醒和可选长休息。
