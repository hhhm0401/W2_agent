# 开发决策与调试记录

## 1. 决策记录

### D01：采用 React + TypeScript + Vite

五选一项目原始建议使用 React 和 TypeScript。该组合足以支持组件化、严格类型、快速构建和单元测试，不需要后端。

### D02：不使用 UI 组件库

参考截图的硬边框、错位阴影和不对称网格具有强定制性。直接使用 CSS 令牌比覆盖通用组件库更简单，包体也更小。

### D03：本地优先

设置、当前任务、计时快照和最近记录都写入 `localStorage`。这符合轻量交付和隐私目标，但不提供跨设备同步。

### D04：完成后由用户手动进入下一模式

自动切换容易让用户错过完成状态，也可能在用户离开设备时启动休息倒计时。当前设计先显示完成，再由用户进入休息或下一轮。

## 2. 调试记录

### BUG-01：TypeScript Node 配置阻止统一构建脚本

- **现象**：`npm run build` 报错 `TS5096`，指出 `allowImportingTsExtensions` 必须配合 `noEmit` 或 `emitDeclarationOnly`。
- **原因**：`tsconfig.node.json` 声明了允许 `.ts` 扩展导入，但没有声明只检查不输出。
- **AI 建议**：在 Node 配置增加 `noEmit: true`，或移除不必要的 `allowImportingTsExtensions`。
- **处理**：在 Node 配置补充 `noEmit: true`，随后继续处理 Vite 与 Vitest 的配置类型冲突。
- **结果**：统一脚本 `npm run build` 完整通过，TypeScript 检查与 Vite 生产构建均成功。

### BUG-02：Vitest 在受限沙箱内无法读取配置

- **现象**：Vitest 启动时报 `Cannot read directory "../../..": Access is denied`，随后无法解析 `vite.config.ts`。
- **原因**：esbuild 加载配置时会检查父目录，受限沙箱禁止访问工作区上层目录。
- **处理**：按环境规范在获得许可后于沙箱外执行同一测试命令，没有修改测试内容。
- **结果**：2 个测试文件、10 个测试全部通过。

### BUG-03：移动端 fullPage 截图出现重复拼接

- **现象**：390px 视口的整页截图在长页面中重复拼接了上方内容。
- **原因**：属于浏览器整页截图的长页面拼接异常，页面 DOM 与宽度指标正常。
- **处理**：改为保存 390 × 844 的真实首屏截图，并单独读取 `scrollWidth` 与 `clientWidth`。
- **结果**：`scrollWidth = clientWidth = 375`，确认没有横向溢出；重拍截图无重复。

### BUG-04：推荐算法只有完成样本

- **现象**：若只在倒计时结束时保存记录，完成率永远为 100%，减少时长建议无法反映中断。
- **原因**：数据采集遗漏了主动放弃轮次。
- **处理**：运行或暂停状态点击重置时，保存 `completed=false`、实际时长与暂停次数。
- **结果**：浏览器连续产生 3 条中断样本后，界面正确建议从 25 分钟降低到 20 分钟。
