# 番茄钟规格说明

## 1. 项目范围

本项目是浏览器端轻量番茄钟，服务于学习和工作专注。它不包含账号系统、云同步、复杂项目管理、社交排行和系统级应用阻断。

## 2. 核心目标与验收标准

| 编号 | 核心目标 | 验收标准 |
|---|---|---|
| G1 | 完成稳定的专注/休息计时 | 开始、暂停、继续、重置和模式切换状态正确 |
| G2 | 提高后台计时准确性 | 以绝对结束时间计算，切回页面后自动校准且不出现负数 |
| G3 | 形成本地记录闭环 | 任务、设置、计时状态、最近记录刷新后仍存在 |
| G4 | 提供可解释的个性化建议 | 三条记录后按规则生成建议，范围 15–50 分钟，用户可拒绝 |
| G5 | 提供鲜明且可用的包豪斯界面 | 桌面与 390px 手机无关键布局错误，核心按钮可键盘聚焦 |

## 3. 功能需求

### FR-01 计时显示

- 显示当前模式、阶段、`MM:SS` 剩余时间与进度。
- 专注默认 25 分钟，休息默认 5 分钟。
- 专注可设置为 15–50 分钟，休息可设置为 5–15 分钟。

### FR-02 状态控制

- `IDLE → RUNNING → PAUSED → RUNNING`。
- 到达结束时间后进入 `COMPLETED`。
- 完成后由用户明确进入休息或下一轮，不自动跳转。
- 重置正在进行的专注时，保存一条未完成记录。

### FR-03 当前任务

- 用户可输入最多 48 个字符的当前任务。
- 当前任务与设置保存在浏览器本机。

### FR-04 历史与统计

- 保存最近 30 条专注记录。
- 显示最近 6 条、今日完成轮次与今日专注分钟数。
- 完成与中断状态必须可区分。

### FR-05 自适应建议

- 少于 3 条记录时不生成建议。
- 最近样本完成率 ≥ 80% 且平均暂停 ≤ 0.5 次，建议增加 5 分钟。
- 完成率 < 60% 或平均暂停 ≥ 2 次，建议减少 5 分钟。
- 其他情况维持当前时长。
- 建议限制在 15–50 分钟。
- 两轮内不重复建议。

### FR-06 今日几何海报

- 用几何形状表现最近完成记录。
- 空状态说明完成第一轮后才开始生长。

## 4. 非功能需求

- NFR-01：所有核心数据仅保存在本机，不主动上传。
- NFR-02：桌面与 390px 手机视口不得出现横向滚动。
- NFR-03：键盘焦点清晰，按钮最小高度 44px。
- NFR-04：尊重减少动态效果设置。
- NFR-05：TypeScript 严格模式通过，生产构建通过。
- NFR-06：推荐与时间工具具有自动化单元测试。

## 5. 边界与异常处理

- 本地数据缺失或损坏时回退默认设置。
- 计时到期后剩余时间固定为 0，不产生负数。
- 完成回调必须去重，避免同一结束时间重复保存记录。
- 计时期间禁用时长滑块。
- 清理浏览器站点数据后记录会丢失，操作手册需明确说明。
