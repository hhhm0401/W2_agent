# 界面与交互设计说明

## 1. 设计目标

界面面向需要快速进入专注状态的学生与轻量办公用户。核心动作必须在首屏完成，不用登录、不打开弹窗、不穿过多层导航。品牌气质采用包豪斯式几何、秩序与功能主义，同时避免把界面做成只有装饰、缺少可用性的“风格练习”。

## 2. 视觉参考

本项目参考用户提供的“声控机械臂课程展示系统”界面截图，提取以下稳定特征：

- 米白色网格纸背景；
- 黑色粗边框与向右下错位的硬阴影；
- 红、黄、蓝三原色作为功能状态与视觉节奏；
- 超粗、紧凑的大标题；
- 英文全大写小标签配合中文正文；
- 方形按钮、明确分区、无圆角卡片与柔和投影；
- 圆、方、斜线等几何构成承担信息表达，而非纯装饰。

用户提供的参考图只用于设计分析，不直接复制原项目内容、文字或功能。

## 3. 设计令牌

| 类型 | 令牌 | 当前值 | 用途 |
|---|---|---:|---|
| 颜色 | `--paper` | `#f3efe6` | 页面及浅色区域背景 |
| 颜色 | `--ink` | `#171410` | 正文、边框、硬阴影 |
| 颜色 | `--red` | `#ed3b2e` | 主操作、专注状态、强调 |
| 颜色 | `--yellow` | `#f6c532` | 计时圆盘、统计、建议 |
| 颜色 | `--blue` | `#2865c4` | 主计时区域、链接与焦点 |
| 颜色 | `--muted` | `#756f63` | 次级说明文字 |
| 边框 | `--rule` | `4px` | 主要容器硬边框 |
| 阴影 | `--shadow` | `8px 10px 0` | 包豪斯错位硬阴影 |
| 网格 | 背景网格 | `40px × 40px` | 建立秩序感与对齐参考 |

字体优先使用 `Century Gothic` 作为几何展示字体，正文使用 `Aptos` 与 `Microsoft YaHei UI`。所有字体均使用系统现有字体，避免依赖外部网络字体导致离线界面变化。

## 4. 信息架构

1. 顶栏：品牌、本地优先状态、日期和节奏设置入口。
2. 主区左侧：产品主张与今日统计。
3. 主区右侧：当前任务、计时器、核心操作和精准计时说明。
4. 下方左侧：根据完成记录生成的“今日几何海报”。
5. 下方右侧：自适应建议和最近专注记录。

桌面端使用不对称双列布局；移动端改为纵向顺序，但不隐藏任何核心功能。

## 5. 交互规则

- 页面加载后，输入任务并点击一次“开始专注”即可进入计时。
- 运行时仅突出“暂停”，次要的“重置”保持浅色。
- 专注结束后明确显示“进入休息”，不自动跳走，避免用户错过完成反馈。
- 设置区采用页面内展开，不使用模态弹窗。
- 计时中锁定时长设置，防止误触改变当前轮次。
- 自适应建议必须展示完成率和平均暂停次数，且允许拒绝。
- 键盘焦点使用蓝色四像素轮廓；按钮最小高度为 44 像素。
- 尊重 `prefers-reduced-motion`，减少动态效果时关闭非必要过渡。

## 6. 响应式策略

- `> 1040px`：主区与下方内容采用双列布局。
- `761–1040px`：计时主区改为单列，下方证据区保留双列。
- `≤ 760px`：全部改为单列；计时圆盘和大标题按视口流式缩放；按钮占满宽度。
- 目标手机验收视口为 `390 × 844`，不得出现横向滚动。

## 7. 与 Figma 的关系

Figma 连接需要具体文件键与节点链接才能读取准确变量。本项目未获得可定位的 Figma 节点，因此不虚构 Figma 参数，而是以用户提供的实际截图为设计依据，并把全部参数集中为 CSS 令牌。若后续提供节点链接，可在不改变组件结构的情况下替换令牌并进行逐项视觉校准。
